两套 API：基本 vs 有序
基本符号表 API
public class ST put(Key key, Value val) // 插入 get(Key key) // 查找 delete(Key key) // 删除 contains(Key key) // 是否存在 isEmpty() // 是否为空 size() // 键值对数量 keys() // 所有键
VS
有序符号表 API（额外方法）
public class ST min() / max() // 最小/最大键 floor(key) / ceiling(key) // 向下/向上取整 rank(key) // 小于 key 的键数 select(k) // 排名第 k 的键 keys(lo, hi) // 范围查询 // rank 和 select 互逆： // key == select(rank(key))
有序符号表的关键洞察：当键可以比较大小时，我们不仅能做 put/get，还能做范围查询、排名、取整等操作 —— 这些在数据库应用中极其常见。
两种实现的演进：从链表到有序数组

从最朴素的链表顺序查找，到利用有序性的二分查找，每一步都在权衡查找与插入的效率。

FrequencyCounter 性能对比：平均每次 put() 的比较次数

API 设计中的 6 个关键约定
💎
rank(key) 返回表中小于 key 的键的数量。这个看似简单的方法是整个有序符号表的核心： 1. get() 用 rank() 定位键的位置 2. put() 用 rank() 找到插入点 3. select(k) 与 rank() 互逆：key == select(rank(key)) 4. ceiling() 直接返回 keys[rank(key)] 5. size(lo, hi) = rank(hi) - rank(lo) + (contains(hi)?1:0) 掌握了 rank()，就掌握了有序符号表的全部秘密。
