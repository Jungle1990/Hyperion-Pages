哈希表 vs 红黑BST：如何选择？
哈希表 (HashST)
// 优势： // 1. 查找/插入平均 O(1) // 2. 代码更简单 // 3. 对标准类型有内置 hashCode() HashST st = new HashST<>(); st.put("key", 42); int val = st.get("key"); // O(1)
VS
红黑BST (ST)
// 优势： // 1. 最坏情况 O(lg N) 有保证 // 2. 支持有序操作：rank, select, // range, min, max, floor, ceiling // 3. 不需要设计 hash 函数 ST st = new ST<>(); st.put("key", 42); st.rank("key"); // 有序操作！
经验法则：大多数程序员默认用哈希表，除非需要有序操作（rank、select、范围查找）时才用红黑BST。 当键是长字符串时，Trie（第5章）比两者都更好。
从 ST 到 SET：当你不需要值的时候

SET 是符号表的退化形式——只关心键是否存在，不关心关联的值。去重、白名单、黑名单都是 SET 的典型应用。

稀疏向量：符号表带来十亿倍加速

当矩阵中 99.99% 的元素为零时，用符号表只存非零元素，矩阵-向量乘法从 O(N²) 降到 O(N + 非零元素数)。

💎
Google 的 PageRank 算法本质上就是反复做稀疏矩阵-向量乘法，直到结果收敛。 网页数 N ≈ 10¹⁰，但每个网页平均只链接到不到 10 个其他网页（极度稀疏）。 用标准数组：N² = 10²⁰ 次运算 → 不可能完成。 用 SparseVector：N × 10 = 10¹¹ 次运算 → 加速十亿倍（10⁹），变为可行！ 「不利用稀疏性的程序员严重限制了自己解决实际问题的能力。」
