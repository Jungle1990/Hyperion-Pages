从暴力到精妙：子字符串查找的进化之路

四种算法代表了四种截然不同的设计思路，每一步进化都是对「如何利用已知信息」的更深思考。

💡
构建 DFA 最精妙的部分是：我们用「部分构建好的 DFA」来计算「还没构建的部分」。维护一个重启状态 X，它代表「如果我们回退并重新扫描已匹配字符，DFA 会到达的状态」。因为 X < j，所以 DFA 在位置 X 的行为已经确定，可以直接复制 dfa[][X] 到 dfa[][j] 作为失配转移，然后覆盖匹配转移为 j+1。这种「自举」构造让整个预处理只需 O(MR) 时间。
暴力法的两种实现：显式回退 vs 隐式回退
版本1：i 指向匹配起点
for (int i = 0; i <= N - M; i++) { int j; for (j = 0; j < M; j++) if (txt.charAt(i+j) != pat.charAt(j)) break; if (j == M) return i; }
VS
版本2：i 指向匹配终点（显式回退）
for (i = 0, j = 0; i < N && j < M; i++) { if (txt.charAt(i) == pat.charAt(j)) j++; else { i -= j; j = 0; } // 回退! } if (j == M) return i - M;
版本2 让回退行为变得显式：失配时 i 需要回退 j 个位置。KMP 的核心贡献就是消除了这个回退——通过 DFA 直接跳到正确的模式位置，i 永远只往前走。这对流式输入（如网络数据）至关重要，因为你无法回退已经读过的数据。
