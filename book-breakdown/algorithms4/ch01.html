API 的本质是一份「契约」：客户端只需要知道方法签名和行为描述，不需要知道内部实现。这带来三个关键好处： 1. 分而治之 -- 大问题拆成小模块 2. 代码复用 -- 写一次，到处用 3. 可替换性 -- 随时换更好的实现（比如把暴力搜索换成二分查找），客户端代码一行不改 这正是本书能不断优化算法的基础设施。

数组的关键陷阱：别名(Aliasing)
你以为的：独立副本
int[] a = {1, 2, 3}; int[] b = a; // b是a的副本？ b[0] = 999; // 你以为 a[0] 还是 1 // 实际上 a[0] 也变成了 999！
VS
正确的复制方式
int[] a = {1, 2, 3}; int[] b = new int[a.length]; for (int i = 0; i < a.length; i++) b[i] = a[i]; // 逐个复制 b[0] = 999; // 现在 a[0] 仍然是 1
Java 中数组名是引用（类似C的指针）。int[] b = a 只是让 b 和 a 指向同一块内存。这个陷阱在传参时更隐蔽：Arrays.sort(a) 会直接修改调用者的数组！
