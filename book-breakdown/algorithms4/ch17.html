有向图 = 单行道
每条路只能单向行驶 「可达」不对称——A 到 B 能走，B 到 A 未必能走 强连通分量需要两遍 DFS（Kosaraju 算法）

拓扑排序三步流水线
从有向图到合法排序
环检测：onStack[] 的精妙设计
仅用 marked[]（错误）
// 只能判断"是否访问过" // 但访问过不代表在当前路径上 if (marked[w]) { // w 已访问，但可能早就 // 从递归栈弹出了 // 不能判定有环！ }
VS
marked[] + onStack[]（正确）
// onStack[v] = true: v在当前递归栈上 onStack[v] = true; marked[v] = true; for (int w : G.adj(v)) if (!marked[w]) { dfs(G, w); } else if (onStack[w]) { // w 在当前路径上 -> 找到环！ // edgeTo[] 可以回溯整个环 } onStack[v] = false; // 退出时弹栈
marked[] 记录「全局是否访问过」，onStack[] 记录「是否在当前 DFS 路径上」。只有当 w 在当前路径上（onStack[w] == true）时，边 v->w 才构成环。这就像迷宫探索中的线团——只有撞到自己还没收回的线，才说明走了一圈。
Kosaraju 算法：用两遍 DFS 找强连通分量

Kosaraju 算法是本章最精妙的算法：代码极简，但正确性证明需要深入理解。关键在于「为什么要先对反向图求逆后序」。

💡
为什么先对反向图求逆后序？想象强连通分量组成的「缩图」（kernel DAG）。如果分量 A 有边指向分量 B，我们需要先处理 B 再处理 A——否则从 A 出发的 DFS 会「越界」到 B。反向图的逆后序恰好给出了缩图的拓扑序的反序，保证每个分量在被处理时不会「漫出」到其他分量。
