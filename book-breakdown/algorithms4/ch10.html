排序算法的应用
数据排好序后： 二分搜索 O(log N)，百万数据也秒查 去重只需线性扫描相邻元素 中位数、百分位数直接按索引取值 Comparator 接口让同一数据支持多种排序规则
从 Comparable 到 Comparator：灵活排序的进化之路

Java 提供两套排序接口：Comparable 定义天然顺序，Comparator 支持任意自定义顺序。理解它们的区别是写出灵活代码的关键。

稳定性：被忽视的关键属性
不稳定排序（如快速排序）
按城市排序后： Chicago 09:25:52 ← 时间乱了! Chicago 09:03:13 Chicago 09:21:05 Chicago 09:19:46 Chicago 09:00:00 同一城市内的时间顺序被打乱， 之前按时间排的结果被破坏了。
VS
稳定排序（如归并排序）
按城市排序后： Chicago 09:00:00 ← 时间仍有序! Chicago 09:00:59 Chicago 09:03:13 Chicago 09:19:32 Chicago 09:21:05 同一城市内的时间顺序保持不变， 先按时间排再按城市排 = 双重有序。
稳定性意味着相等键的元素保持原来的相对顺序。Java 的 Arrays.sort() 对对象用归并排序（稳定），对基本类型用快速排序（快但不稳定）——这是深思熟虑的设计决策。
归约思维：用排序解决「非排序」问题
排序是通用的问题归约器

⚡
找第 k 小的元素不需要完整排序！select() 借用快速排序的 partition()，每次只处理包含目标索引的那一半。 期望比较次数 ≈ 2N + 2k·ln(N/k) + 2(N-k)·ln(N/(N-k))，对于中位数（k=N/2）约为 (2+2ln2)N ≈ 3.39N 次比较。 类比：在一堆试卷中找中等分数的那张，不需要把所有试卷从高到低排好——只需要不断把试卷分成「高于」和「低于」某个基准的两堆，直到目标试卷浮出水面。
