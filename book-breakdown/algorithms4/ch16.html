图的三种表示法：该选哪个？
邻接矩阵
// V x V 布尔矩阵 boolean[][] adj = new boolean[V][V]; // 添加边 v-w adj[v][w] = true; adj[w][v] = true; // 空间: V^2 (百万顶点 = 万亿空间!) // 检查邻接: O(1) // 遍历邻居: O(V)
VS
邻接表 (本章选择)
// V 个链表/Bag Bag[] adj = new Bag[V]; // 添加边 v-w adj[v].add(w); adj[w].add(v); // 每条边存两次 // 空间: V + E (稀疏图友好!) // 添加边: O(1) // 遍历邻居: O(degree(v))
真实世界的图几乎都是稀疏图(边数 E 远小于 V^2)。邻接表在空间和遍历效率上都是最优选择。 邻接矩阵唯一的优势是 O(1) 判断两点是否相邻，但这个操作在大多数算法中并不需要。
从 DFS 到 BFS：搜索策略的演进

DFS 和 BFS 的代码框架几乎相同，唯一的区别在于数据结构：栈 vs 队列。这个微小的变化带来完全不同的搜索行为。

DFS vs BFS：同一个图，两种视角
DFS (深度优先)
数据结构: 栈 (LIFO / 递归调用栈) 搜索策略: 一条路走到黑, 碰壁再回头 路径特点: 长且曲折 适用场景: 连通性检测, 环检测, 二分图判定 类比: 一个人独自探索迷宫 手持线团, 走到死胡同就回退
VS
BFS (广度优先)
数据结构: 队列 (FIFO) 搜索策略: 层层推进, 由近及远 路径特点: 最短路径 适用场景: 最短路径, 分离度问题 类比: 一群人同时扇形展开搜索 每到岔路口就分头行动
两者的通用框架完全相同——把源点放入数据结构，然后循环取出顶点、标记、把未标记的邻居放入数据结构。 唯一区别：取出规则是 LIFO(DFS) 还是 FIFO(BFS)。这个微小的差异导致了完全不同的搜索行为。
🧩
连通分量(CC)将图划分为若干「互不相连的子图」。DFS 的实现极其优雅： 对每个未标记的顶点启动一次 DFS，每次 DFS 能到达的所有顶点就构成一个连通分量。 预处理后，判断任意两点是否连通只需 O(1) 时间——只要比较它们的分量编号 id[v] == id[w]。 与 Union-Find 相比：DFS 预处理后查询是常数时间(更快)，但需要先构建完整的图； Union-Find 是在线算法，可以边加边边查询，不需要提前知道所有边。各有优势。
