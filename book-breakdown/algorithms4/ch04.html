同一个问题，不同的增长阶数带来天壤之别的性能。关键不是「优化常数」而是「降低阶数」。

增长阶数对比：N=10000 时的操作次数

🌊
N(N-1)(N-2)/6 = N^3/6 - N^2/2 + N/3 当 N=1000 时，N^3/6 ≈ 1.667 亿，而 -N^2/2 + N/3 ≈ -49.97 万，仅占 0.03%！ 所以我们大胆写成 ~N^3/6，只保留最高阶项。这就是「波浪号近似」的威力 —— 让复杂的数学公式变简单，而误差随 N 增大趋近于零。

推理过程
比值趋近 8 = 2^3 → 增长阶数是 N^3 预测 N=16000: 51.1 × 8 = 408.8 秒 实际: 409.3 秒 ✓ T(2N)/T(N) ~ 2^b → b = lg(ratio)
倍率实验是「穷人的算法分析」—— 不需要数学推导、不需要了解内部实现，只要翻倍输入、记录时间、算比值。比值稳定后，你就知道了增长阶数。任何程序都适用！
内存分析速查表：Java 对象到底占多大？
📊
ResizingArray 的 push()：大多数时候只访问 1 个数组元素，但偶尔需要扩容 —— 拷贝整个数组！ N 次 push 的总数组访问次数：N + 4 + 8 + 16 + ... + 2N = 5N - 4 均摊下来每次 push 只要约 5 次数组访问，是常数级别！ 关键思想：用大量「便宜操作」的盈余来「补贴」偶尔的「昂贵操作」，看总账而非单笔账。
