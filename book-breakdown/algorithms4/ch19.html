GPS 导航
交叉口 = 顶点，道路 = 有向边 通行费/时间 = 权重 从你当前位置出发，找到去每个目的地最便宜的路线 把所有最优路线画出来，形成一棵「导航树」
⟺
🖥️
SPT 算法
顶点对应交叉口，有向边对应道路 边的权重对应代价（距离/时间/费用） 从源点 s 出发，对每个可达顶点计算最低代价路径 所有最短路径构成一棵「最短路径树 (SPT)」
🔑
「松弛」是所有最短路径算法的基础操作。对于边 v->w：如果经由 v 到达 w 的路径更短（distTo[v] + weight < distTo[w]），就更新 distTo[w] 和 edgeTo[w]。 想象一根橡皮筋绷在两个钉子之间 —— 如果发现了更短的路径，就把橡皮筋「松弛」到新路径上。 所有算法的区别仅在于：按什么顺序松弛哪些边。
三大算法的代码演进

从 Dijkstra 到拓扑排序法再到 Bellman-Ford，核心都是 relax()，区别在于松弛边的顺序策略。

三大算法对比：何时用哪个？
Dijkstra 算法
前提：权重 >= 0 时间：O(E log V) 空间：O(V) 策略：贪心 + 索引优先队列 适用：路网导航、正权图
VS
Bellman-Ford 算法
前提：无限制（可检测负环） 时间：O(EV) 最坏情况 空间：O(V) 策略：多轮松弛 + FIFO 队列 适用：汇率套利、带截止期的调度
DAG 最短路径（AcyclicSP）是第三个选择： 前提：图必须无环（但允许负权边） 时间：O(E + V) —— 三者中最快！ 策略：拓扑排序 + 一轮松弛 适用：项目调度（关键路径法）、DAG 中的最长路径
关键路径法 (CPM)：最短路径的经典应用
将并行作业调度问题转化为 DAG 最长路径问题
💰
把汇率表建模为有向完全图，边权 = -ln(汇率)。 乘法路径（1000 USD × 0.741 × 1.366 × 0.995）变成加法路径（-ln(0.741) + -ln(1.366) + -ln(0.995)）。 如果某条环路的权重之和为负 => 乘积 > 1 => 套利机会！ 1000 美元 → 741 欧元 → 1012.206 加元 → 1007.14 美元，净赚 7.14 美元。 Bellman-Ford 的负环检测就是交易员发现套利机会的「秘密武器」。
