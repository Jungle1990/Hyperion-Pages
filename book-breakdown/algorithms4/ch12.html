BST 核心操作的递归之美

BST 的 get/put/delete 都遵循同一个递归模式：比较 -> 选择方向 -> 递归到子树 -> 回溯时更新。

select() 与 rank()：一对互逆操作
select(k) -- 找排名第 k 的键
private Node select(Node x, int k) { if (x == null) return null; int t = size(x.left); // 左子树大小 if (t > k) return select(x.left, k); else if (t < k) return select(x.right, k-t-1); else return x; // 就是当前节点 }
VS
rank(key) -- 找键的排名
private int rank(Key key, Node x) { if (x == null) return 0; int cmp = key.compareTo(x.key); if (cmp < 0) return rank(key, x.left); else if (cmp > 0) return 1 + size(x.left) + rank(key, x.right); else return size(x.left); }
两者都依赖每个节点维护的子树大小 N。select 类似快速排序的 partition 选择：左子树有 t 个节点，如果 k < t 就在左边找，k > t 就在右边找，k == t 就是当前节点。rank 是 select 的逆运算。
BST 性能：最好、平均、最坏

💡
BST 和快速排序本质上是同一个算法的两种表达形式！ 根节点 = 第一个 partition 元素：左子树所有键都小于根，右子树所有键都大于根。 子树递归构建 = quicksort 的递归子数组排序。 这就是为什么随机 BST 的搜索成本 ~2lnN 和 quicksort 的比较次数 ~2NlnN 有相同的 2lnN 因子。 同样，如果输入已排序，quicksort 退化为 O(N^2)，BST 退化为链表 —— 根源都是 partition 元素选得太极端。
