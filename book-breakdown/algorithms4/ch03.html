栈 (Stack) vs 队列 (Queue)
栈 = 叠盘子：最后放上去的最先拿走 (LIFO) 队列 = 排队：最先到的最先离开 (FIFO) API 几乎相同，但「谁先出去」的规则完全相反，导致截然不同的行为
栈的四次进化：从玩具到工业级

从最简单的固定容量字符串栈，逐步演化为支持泛型、动态扩容、可迭代的通用栈实现。

数组 vs 链表：两种根本不同的数据组织方式
数组 (顺序分配)
// 优势：索引直达，O(1) 随机访问 int x = a[i]; // 劣势：大小固定，扩容要搬家 Item[] temp = new Object[2 * N]; for (int i = 0; i < N; i++) temp[i] = a[i]; // 全部复制!
VS
链表 (链式分配)
// 优势：插入删除 O(1)，空间按需分配 Node oldfirst = first; first = new Node(); // 轻松插入 // 劣势：找第 k 个元素要从头遍历 for (Node x = first; x != null; x = x.next) // 只能一个一个走...
这两种数据组织方式是所有复杂数据结构的基础。后续章节中的二叉树、哈希表、图等都是它们的组合与扩展。选择哪种取决于你的操作模式：频繁随机访问选数组，频繁插入删除选链表。

没有泛型的世界：你需要为 String 写一个栈，为 Integer 写一个栈，为 Transaction 写一个栈……每个都是重复代码。 有了泛型：Stack 中的 Item 是「类型占位符」，客户端用 Stack 或 Stack 时，编译器自动替换并做类型检查。 代价：Java 泛型不支持基本类型（int/double），需要通过自动装箱/拆箱（autoboxing）桥接。stack.push(17) 会自动把 int 转为 Integer。
迭代器模式：让 foreach 成为可能
🚨
pop() 后，被弹出的元素虽然对客户端已不可见，但数组槽位仍持有它的引用。Java 垃圾回收器无法回收这个「孤儿对象」，因为它仍被数组引用着。 修复极其简单：a[N] = null; 一行代码，把废弃引用清空。 这不是性能优化，而是正确性问题 —— 在长期运行的系统中，对象游离会导致内存泄漏。
