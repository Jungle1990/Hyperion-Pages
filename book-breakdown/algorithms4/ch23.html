正则表达式的三个基本操作
操作定义
// 1. 连接 (Concatenation) AB → {"AB"} // A后面接B // 2. 或 (Or) A|B → {"A", "B"} // A或者B AB|BCD → {"AB", "BCD"} // 连接优先级更高 // 3. 闭包 (Closure) A* → {"", "A", "AA", "AAA", ...} // 重复0次或任意次 (AB)* → {"", "AB", "ABAB", ...}
VS
优先级 (高→低)
// 优先级从高到低： // 1. 闭包 * (最高) // 2. 连接 (中间) // 3. 或 | (最低) // 例子： AB* → A后面接0或多个B {"A", "AB", "ABB", ...} (AB)* → 0或多个"AB"整体 {"", "AB", "ABAB", ...} // 括号可以覆盖默认优先级
和算术表达式类似：* 像乘法优先于 + 一样，闭包优先于连接优先于或

NFA 模拟：从概念到代码

NFA 模拟的核心是维护一个「可能状态集合」，每读一个字符就更新这个集合。

NFA 构造：用栈翻译正则表达式

NFA 构造器像 Dijkstra 的双栈算法一样，用一个栈跟踪左括号和 | 的位置，逐字符构建 ε-转换有向图。

正则表达式实战速查
💡
Kleene 定理证明了正则表达式和 NFA 的等价性：每个 RE 都有对应的 NFA，反之亦然。这意味着一种「描述语言」和一种「计算机器」在表达能力上完全等价 —— 这是理论计算机科学最优美的结果之一。我们的算法正是这个定理的构造性证明：给定任意 RE，我们确实能构造出对应的 NFA。
