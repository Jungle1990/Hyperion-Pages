二叉堆
新元素插到末尾 → 通过 swim 上浮到正确位置 取最大值 → 从堆顶取出，末尾补上再 sink 下沉 修改优先级 → swim 或 sink 恢复堆序 两个操作都保证 O(log N)
swim 与 sink：堆的两个核心操作

swim（上浮）处理「节点比父亲大」的违规，sink（下沉）处理「节点比孩子小」的违规。所有堆操作都建立在这两个原语之上。

三种实现的性能对决
基础实现
无序数组： insert = O(1) // 直接追加 delMax = O(N) // 遍历找最大 有序数组： insert = O(N) // 插入排序式移位 delMax = O(1) // 直接取末尾
VS
二叉堆实现
二叉堆： insert = O(log N) // swim 上浮 delMax = O(log N) // sink 下沉 两个操作都快！ 代价：需要维护堆序性质
基础实现总有一个操作是 O(N)，二叉堆两个都是 O(log N)。当 N 很大且 insert/delMax 交替频繁时，堆的优势是压倒性的。对于 N=10 亿，log N 仅约 30！
🧮
二叉堆的精妙在于：用数组存完全二叉树，不需要任何指针！ 位置 k 的父节点 → k/2 位置 k 的左孩子 → 2k 位置 k 的右孩子 → 2k + 1 pq[0] 不用，堆从 pq[1] 开始。N 个节点的堆高度为 floor(lg N)。 这种紧凑表示让缓存更友好，操作更快——零指针开销，纯粹靠下标计算导航整棵树。
