Quick-Find vs Quick-Union：两种极端策略
Quick-Find (扁平数组)
id[]: [1, 1, 1, 8, 8, 1, 1, 1, 8, 8] find(p): return id[p] → O(1) union(p,q): 扫描全部 id[] 改值 → O(N) 优点：查询极快 缺点：每次合并代价太高
VS
Quick-Union (树/森林)
id[]: [1, 8, 1, 8, 3, 0, 5, 1, 8, 8] ↑ 父节点链接，形成树结构 find(p): 沿链上溯到根 → O(树高) union(p,q): 改一个根的链接 → O(1)+find 优点：合并只改一条边 缺点：树高不受控，可退化
两种策略各走一个极端：Quick-Find在find上极致，Quick-Union在union上极致。加权Quick-Union的天才在于：仅用一个小规则（小树挂大树）就同时约束了两者。
🧮
用归纳法证明：大小为 k 的树高度最多 lg k。 当大小为 i 的树合并到大小为 j 的树（i ≤ j）时，被合并树中所有节点深度 +1，但它们现在属于大小 i+j 的树。 因为 i ≤ j，所以 i+j ≥ 2i，即 1 + lg i ≤ lg(2i) ≤ lg(i+j)。 每个节点的深度只在它所在的树被合并为「小树」时才会增加，而每次增加后树的大小至少翻倍——所以最多增加 lgN 次。
