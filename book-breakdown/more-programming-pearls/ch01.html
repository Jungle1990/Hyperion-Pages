"一个程序中不到 4% 的语句通常占用了一半以上的运行时间" 这意味着：你不需要优化整个程序，只需要找到那 4% 就够了！Profiler 就是帮你找到它们的工具。
实战案例：素数程序的 6 次进化

Bentley 通过 Profiler 逐步将一个简单的素数计算程序优化了100倍，每一步都是 Profiler 指引方向。

运行时间对比 (n=1000, 相对于 P1 基准)

🎯
P2 → P1 的逆转是本章最重要的一课：减少代码的"执行次数"不等于减少"运行时间"。 P2 的因子测试次数从 78,022 降到 5,288（减少14倍），但因为每次都调用昂贵的 sqrt()，反而更慢了。 没有 Profiler，你可能永远猜不到 sqrt() 才是元凶。量化 > 直觉。

自己造一个 Profiler！
原始代码
for (i = 3; i <= n; i++) { if (prime(i)) { print i } }
VS
插桩后的代码
for (i = 3; i <= n; i++) { <<<998>>> if (prime(i)) { <<<167>>> print i } }
Bentley 仅用两个 5 行的 Awk 程序就实现了一个基本的行计数 Profiler： 程序1: 在每个 { 后面插入计数器代码 → 程序2: 将计数结果合并回源代码显示 总共不到几小时的工作量
