Step 1
问题定义

在包含 N 个元素的无序数组中，找到第 K 小的元素。典型应用：找中位数（K = N/2）、找百分位数、找前 K 小的元素集合。

// 朴素方法：排序 → O(N log N)
Sort(X[1..N])
return X[K]

// 能否做到 O(N)？答案是肯定的！
// Hoare 的选择算法：平均只需 ~3.4N 次比较找到中位数

选择算法可视化

交互式体验 Hoare 选择算法的每一步。选择 K 值，然后逐步观察算法如何通过分割缩小搜索范围。

18 个元素的数组，可调节 K 值（1-18），逐步演示分割过程，显示比较次数、分割轮次和当前范围大小。

Step 4
「砍一半」的力量：N + N/2 + N/4 + ... ≤ 2N

每次分割后，只需要在包含 K 的那一半继续搜索。平均而言，每次搜索范围减半。

N=128 时: 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255 ≤ 2×128 = 256

总工作量 ≤ 2N，所以选择算法是 O(N)。

性能对比
排序 vs 选择

对于不同规模的 N，观察全排序 O(N log N) 与选择 O(N) 的耗时对比。100N 微秒 vs 100N·log₂N 微秒的实际差距。

N排序 O(N log N)选择 O(N)速度比
1,000~1 ms~0.34 ms~2.9x
10,000~13 ms~3.4 ms~3.9x
100,000~170 ms~10 ms~17x
1,000,000~2 s~0.34 s~5.9x
