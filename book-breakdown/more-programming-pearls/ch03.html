当 l=4, u=5 时，m = int((4+5)/2) = 4。 此时 x[4]=40 < 50，执行 l = m = 4 —— l 没有变化！ 下一轮仍然是 l=4, u=5, m=4，永远循环。 修复：l = m + 1 保证了 l 至少前进 1 步，搜索范围一定会缩小。
Selection 算法：尾递归消除的陷阱
Buggy: 递归转迭代时的错误
# 原始递归版本（正确）： function select(l, u, k) { if (l >= u) return swap(l, rand(l,u)) # 随机选 pivot m = l for (i = l+1; i <= u; i++) if (x[i] < x[l]) swap(++m, i) swap(l, m) if (m < k) select(m+1, u, k) else select(l, m-1, k) }
VS
Fixed: 正确的迭代版本
# 消除尾递归 → 迭代： function select(l, u, k) { while (l < u) { swap(l, rand(l,u)) m = l for (i = l+1; i <= u; i++) if (x[i] < x[l]) swap(++m, i) swap(l, m) if (m < k) l = m + 1 else u = m - 1 } }
原始 buggy 版本中 u = m（而非 u = m-1），但由于 swap(l, rand(l,u)) 的随机性，错误输出看起来"几乎正确" —— 随机 pivot 选择掩盖了 bug，让错误只在特定情况下出现。 脚手架的价值：通过大量自动化测试（如 1000 次随机输入），脚手架能可靠地暴露这类概率性 bug。
