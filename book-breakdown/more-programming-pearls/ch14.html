// Program 1: 朴素实现
float dist(A[], B[], K) {
    Sum = 0;
    for (i = 0; i < K; i++)
        Sum += (A[i]-B[i]) * (A[i]-B[i]);
    return sqrt(Sum);  // <-- 瓶颈在这!
}

「好起点」的秘密 -- 为什么选 2D?

真实距离在 D 和 √K·D 之间 (D = max|A[i]-B[i]|)。选 2D 作为起点，因为它接近几何均值 √(D·√K·D)。

以 K=4 为例: D = max 差值

D --- 2D(起点) --- 真实距离 --- √K·D

起点越接近真实值，需要的迭代次数就越少。2D 通常只需 4 次迭代即可收敛。

算法 vs 硬件: 世纪竞赛

3D 椭圆偏微分方程 (N×N×N 网格) -- 30 年的进步

算法改进 (250,000x)
1945Gauss-Seidel 迭代 O(N⁵) -- 基准
1954SOR 超松弛 O(N⁴) -- 快 N 倍
1971FFT 快速傅里叶 O(N³ log N) -- 又快 N/logN 倍
1978Multigrid 多重网格 O(N³) -- 最优! 与网格同阶
硬件改进 (500,000x)
1945ENIAC 真空管 5,000 次/秒
1955晶体管计算机 ~100x 提升
1970集成电路时代 ~10,000x 提升
1985VLSI / 超大规模集成 500,000x 总提升

250,000 x 500,000 = 1.25 x 10¹¹

算法 + 硬件的双重进步: 曾经需要数个世纪的计算，现在不到 1 秒。

软件 30 年提升 250,000x ≈ 硬件 40 年提升 500,000x。算法的力量不可小觑!
